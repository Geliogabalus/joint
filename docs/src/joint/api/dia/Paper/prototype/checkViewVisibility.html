paper.checkViewVisibility(cellView)

For the specified view, calls the visibility viewport function specified by the paper.options.viewport function. If the function returns true, the view is attached to the DOM; in other case it is detached.

While async papers do this automatically, synchronous papers require an explicit call to this method for this functionality to be applied. To show the view again, use paper.requestView(). If you are using autoFreeze option you should call this function if you are calling paper.requestView() if you want paper.options.viewport function to be applied.

paper.checkViewVisibility(cellView, opt)

If opt.viewport is provided, it is used as the callback function instead of paper.options.viewport. The format of the callback method is described in the option's documentation, as are examples of usage.

In async papers, providing opt.viewport causes view visibility to be temporarily recalculated according to the provided callback function.
